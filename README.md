# イニシャルT

three.js の平面ワールドを、MagicaVoxel (.vox) 形式の車で走り回るゲームです。

- **Toyota 86 / Volvo 240** … タイトル画面で選択するプレイヤー操作車
- **CPU 車** … 起動時に `vox/` 直下を自動確認し、選択中のプレイヤー車以外を使用
  (kabu, nissan, 180sx, toyotahigh, probox, vw, volvo_sedan …)。街(`?map=city`)
  では全車種が道路を左側通行で走り回る。日本橋ではCPU車を配置しない
- **vox/object/tree01.vox / tree02.vox** … フィールドにランダム配置される樹木(当たり判定あり)

`vox/` 直下は車両専用、コース上の樹木などは `vox/object/` に置きます。

## デモ画面

読み込みが終わると、まず**デモ画面**になります。

- ユーザー車が自動運転で道路を回遊し、コーナーで積極的に**ドリフト**します
- ワンダーランドでは、街中・外周道路・森林コース・峠コースからデモ場所と開始地点を毎回ランダムに選びます
- 20秒に1度、5秒ほどカメラ視点がランダムに切り替わります
- 画面中央に白文字の **Demo** が点滅表示されます
- **何かキーを押す(または画面をクリック)とゲーム開始**

## 街路(自動生成)

- 縦横の格子道路 + たまに斜めの道路(シード値で決定的に生成)
- 街の外周は環状道路で接続され、縦横・斜め道路の行き止まりがなく循環できる
- 道路は2種類: **片側二車線(計4車線・幅14m)** と **片側一車線(計2車線・幅8m)**
- 白線は日本方式: 外側線=実線、4車線の中央線=実線+車線境界=破線、2車線の中央線=破線
- 交差点内は白線なし(素のアスファルト)
- 区画(ブロック)には大きさに応じて**約9〜25棟**の建物を高密度配置。
  建物の前面は最寄りの道路側を向く
- 建物は現在プレースホルダーの箱。`js/main.js` の `BUILDING_VOX` に .vox ファイル
  (前面 = モデルの +Z)を追加すると、箱と混ざってランダムに配置される

## 森のコース(郊外)

- 街の東側の郊外に、多数の連続カーブを持つ**2車線の周回コース**を自動生成
- コース沿いには `tree01` / `tree02` の樹木を約756個配置する
- 樹木は道路の左右約9〜27mに集中し、道路から離れた場所には原則配置しない
- 街の東端の道路から連絡路でつながっている
- AI 車が1台このコースを周回している

## ドリフトコース(街の南)

- 街の南側に、左右2ブロックで**ヘアピン(Uターン)が連続する高密度な蛇行コース**を用意
- 直線は約80mと短く、合計14回のタイトな180°ターンが続くため、
  サイドブレーキ(SPACE)で**連続ドリフト**しやすい
- 左側の連続ヘアピンを登り、約24mの標高差がある頂上を越えて、右側の連続ヘアピンを
  下る**峠型レイアウト**。道路も周囲の地面も勾配に沿って上下する
  (街のグリッド道路は従来どおり平坦)
- グリッド道路から短い連絡路でつながっている。CPU 車も2台走っている
- **デモ画面ではこのコースをドリフトで周回**する(何かキーでゲーム開始)

CPU 車は `vox/` 内の全車種(約28種)。すべて**左側通行**で、街路の巡回コースや
森のコースを勝手に走っている。処理を軽く保つため、車種ごとにジオメトリは1つだけ持ち、
CPU 車のシャドウマップ描画は省いている(接地影は残る)。
車両 `.vox` の追加・削除時に `js/main.js` の車種一覧を編集する必要はない。

## 信号

- 格子道路の全交差点に信号機(ポール+横型三灯、日本式の 青・黄・赤)を設置
- サイクル: **青8秒 → 黄3秒 → 赤11秒**(赤の間は交差方向が青8秒+黄3秒)
- 交差点に近づくと、画面上部に**自分の進行方向の信号がウインドウ表示**される
- **CPU車は信号を守る**: 赤・黄なら停止線で止まる(黄で止まれない距離なら通過)。
  プレイヤーは自由(信号無視も可能)

## 緊急指令(ミッション ※ワンダーランドのみ)

- **プレイ開始の1分後**に、画面中央へ**黒背景・白文字のポップアップで緊急指令**が出る
  (例:「本部より緊急指令!管内で銀行強盗が発生した。逃亡中の白い日産180SXを追跡、確保せよ。」)
- 指令された**犯人車両が、毎回ランダムに選ばれるコース(外周・区画・環状・森林)を高速で逃走**する。上部に「🚨 追跡中: ◯◯」を常時表示
- **犯人車両に体当たり**すると「犯人確保!」で解決。**解決の1分後に次の指令**が出る
- 逃走車は**直線最高速度108km/h・加速はユーザー車の約1.5倍**で、信号を無視して逃げる
- 指令はランダム順で、**全件(既定5件)出たら終了**
- 指令セットは **`ゲームシナリオ.txt`** に「車両ファイル + 指令メッセージ」で記述。
  行を足せば指令を追加・並べ替えできる(犯人車両は `vox/` 直下の .vox 名を指定)
- 日本橋マップは CPU 車が居ないため、この指令は発動しない

## 遊び方

ブラウザのセキュリティ制限 (fetch) のため、ローカルサーバー経由で開いてください。

### Windows（かんたん起動）

`PLAY_VOX_DRIVE.bat` をダブルクリックしてください。ローカルサーバーが起動し、
ゲームがブラウザで自動的に開きます。プレイ中はサーバーのウィンドウを開いたままにし、
終了するときはそのウィンドウを閉じてください。

このBATと `local-server.ps1` は、リポジトリをダウンロードして遊ぶ場合専用です。
GitHub Pages上のゲームでは使用せず、GitHubの `vox/` フォルダを直接確認します。

自動生成の街 + 森 + 信号マップで遊ぶ場合は、`PLAY_CITY_MAP.bat` を
ダブルクリックしてください。

PowerShell の標準機能だけを使うため、Python や Node.js の追加インストールは不要です。

### コマンドで起動

```sh
python3 -m http.server 8000
# → http://localhost:8000 を開く
```

## 操作

| キー | 動作 |
| --- | --- |
| `A` | ブレーキ |
| `S` | アクセル |
| `↑` | シフトアップ |
| `↓` | シフトダウン |
| `←` `→` | ハンドル |
| `SPACE` | サイドブレーキ(ドリフト) |
| マウスドラッグ | カメラ視点 |
| ホイール | ズーム |
| `M` | 真上から地図表示（全マップ共通。自車=白丸、追跡中の犯人車=赤丸をリアルタイム表示） |
| `N` | サウンド ON/OFF |

ギアは `R / N / 1〜5` のマニュアル式。各ギアに最高速度があるので、上手にシフトチェンジして走ってください。

ユーザー車の最高速度は全マップ共通で180km/hです。4速は100km/hまで、5速は100〜180km/hを担当します。鈴鹿では、ユーザー車とは異なる10台のCPU車が90〜110km/h設定（設定平均99.2km/h）で周回します。車種は毎回多様に選ばれ（同一車種は色違い含め最大2台、Kabuは最大1台）、軽トラばかりにならないようにしています。

スピードに乗った状態で `SPACE` + ハンドルでテールが流れてドリフトします。滑っている間はスキッドマークとタイヤスモークが出ます。カウンターステアで姿勢をコントロールしてください。

サウンドは Web Audio API によるリアルタイム生成(音声ファイル不使用)。エンジン音は燃焼パルス列に近いリッチな倍音の波形に、**回転で動かない固定フォルマント(共鳴)**を掛けてエンジン固有の「声色」を出しています。低回転ほどドコドコと脈動し、高回転で滑らかな唸りに変化。スロットルを踏むと倍音が開いて硬い音になります。ギアごとに音域が決まっていて回転数と共にピッチが上がり、5速は高音になりすぎないよう4速と同じ音域です。ドリフトや急ブレーキではタイヤのスキール音が鳴ります。

## マップ

ルートURLではタイトル画面が開き、**「Toyota 86」または「Volvo 240」**を選んでから、
**「ワンダーランド」「日本橋」「鈴鹿」**を選択できる。URL パラメータで直接開始することもできる:

```
http://localhost:8000/                        … タイトル / マップ選択
http://localhost:8000/?map=city&car=toyota86            … ワンダーランド / Toyota 86
http://localhost:8000/?map=city&car=volvo240             … ワンダーランド / Volvo 240
http://localhost:8000/?map=nihonbashi.gltf&car=toyota86  … 日本橋 / Toyota 86
http://localhost:8000/?map=nihonbashi.gltf&car=volvo240  … 日本橋 / Volvo 240
http://localhost:8000/?map=suzuka&car=toyota86           … 鈴鹿 / Toyota 86
http://localhost:8000/?map=suzuka&car=volvo240           … 鈴鹿 / Volvo 240
http://localhost:8000/?map=maps/sample.glb   … 任意の glTF/GLB マップ
```

同梱の `maps/sample.glb` が命名規則のサンプル。`nihonbashi.gltf` のような
変換ツール出力の生データ(数千メッシュ・mm単位・Z-up)も自動補正して読み込める:

- **メッシュ統合**: 200個を超えるメッシュはマテリアル別に自動マージ(描画コール削減)
- **単位の自動判定**: 巨大な数値(4000超)なら mm とみなし、さらに走りやすいよう
  **10倍に拡大**して読み込む(実質 1/100 スケール)
- **上軸の自動判定**: Z-up モデルは自動で起こす
- **位置合わせ**: マップ中心を原点へ、**一番低い面(道路・地表)を y=0 へ**
- **路面追従**: 車の直上からのレイキャストで路面の高さに追従(橋や坂も走れる。
  屋根には乗らない)
- URL パラメータで上書き可: `&scale=` `&zup=0/1` `&y=` `&sx=` `&sz=`(スポーン位置)

### マップの命名規則

| 名前 | 意味 |
| --- | --- |
| `col_` で始まるメッシュ | 当たり判定あり(バウンディングボックスから円形コライダーを生成) |
| `spawn` という名前の空オブジェクト | プレイヤーの開始位置(+Z 方向が初期の向き) |
| `wp_<ループ名>_<番号>` の空オブジェクト | AI 車の巡回経路。番号順に周回する(ループは最大4つ、1ループにつきAI車1台) |

### 注意

- 走行物理は平面(y=0)前提。マップの地面は y=0 付近の平らな面にする
- マップには地面を含めること(保険として薄い草地が y=-0.08 に敷かれる)
- カスタムマップでは自動生成の街・森・信号は無効になる
- 単位はメートル(車は全長約 4.8m)

## 構成

- `index.html` — エントリ + HUD(importmap で ES Modules を解決)
- `js/vox.js` — 自作 .vox パーサー(SIZE/XYZI/RGBA + nTRN/nGRP/nSHP シーングラフ対応)。面カリング済みのマージ済みジオメトリを生成
- `js/audio.js` — Web Audio によるエンジン音・タイヤ音の生成
- `js/main.js` — ゲーム本体(車の物理、街の自動生成、AI、信号、カメラ)
- `lib/three.module.js` — three.js r152(ES Module、同梱)
- `lib/GLTFLoader.js` / `lib/BufferGeometryUtils.js` — glTF 読み込み用(同梱)
- `vox/` — ボクセルモデル
- `maps/` — カスタム glTF マップ置き場(`sample.glb` 同梱)
